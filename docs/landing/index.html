One audio source.
Every speaker in the house.

SyncCast captures macOS system audio via ScreenCaptureKit and fans it out to local speakers and AirPlay 2 receivers — in sync, with no driver, no root, no fuss.

Get on GitHub View install

Built for the way you actually listen

macOS gives you two half-solutions. SyncCast unifies them in one menubar.

Whole-home AirPlay sync

Send the same stream to every AirPlay 2 receiver in your home — HomePods, Apple TVs, third-party speakers — locked to PTP for tight cross-room timing.

Stereo local mix

Keep video sync intact for low-latency local playback. A separate Stereo mode routes the built-in speakers and any USB or HDMI outputs through one aggregate device.

ScreenCaptureKit native

Apple's first-class capture API — no virtual driver to install, no kernel extension, no kext signing. SyncCast runs entirely in user-space.

Open source, hackable

MIT-licensed Swift + Python. Pluggable transport layer for AirPlay 2 today and other protocols tomorrow. Read the code, file an issue, send a PR.

How it works

Capture once. Route many. Mode switching is mutually exclusive — Whole-home or Stereo, never both.

Install in three commands

Clone, build, install. No package manager wrappers, no installers — just the source you can read.

# 1. Clone the repo
git clone https://github.com/vcxzvfe/syncast.git
cd syncast

# 2. Build the Swift menubar app + Python AirPlay sidecar
./scripts/bootstrap.sh
./scripts/package-app.sh

# 3. Install to /Applications
bash scripts/install-app.sh

# Launch
open /Applications/SyncCast.app
